# 需求规格说明书

## 墨香书阁

---

## 一、引言

### 1.1 编写目的

本文档从宏观层面描述墨香书阁V3.0.0企业级小说阅读网站的功能需求和非功能需求，为后续设计和开发提供统一的需求基准。

### 1.2 产品概述

本系统是一个面向广大文学爱好者的在线小说阅读与社区互动平台，采用双主题UI设计体系（首页深蓝渐变主题配书库金色暖纸质感），致力于提供高品质的阅读体验和完善的社区互动环境。用户端涵盖首页推荐、书库浏览、详情阅读、关注作者、评分系统、签到会员、个人中心七大模块；管理端涵盖图书管理、分类管理、订单管理、签到管理和粉丝列表五大功能域。技术栈采用前端Vue 3 + TypeScript + Vite + Element Plus，后端Python 3.8 + Django 4.2 + DRF + MySQL，认证方式为Session-based Cookie withCredentials。当前数据规模为81本小说共2381章节，涵盖12个分类。

### 1.3 用户群体

本系统面向以下用户群体：网络文学普通读者（需要便捷的书籍发现和舒适的阅读体验）、注册会员（可享受收藏管理、阅读进度跟踪、关注作者、评分互动、签到会员等个性化服务）、业余网络作者（需要创作发布工具和读者反馈渠道）、平台管理员（需要后台内容管理和运营维护能力）。用户需具备基本的计算机操作能力。

## 二、功能需求

### 2.1 首页模块

作为平台的入口页面，向用户展示热门内容和快捷导航入口，采用深蓝渐变主题设计（#2563EB主色）。

**功能清单**：

| 功能名 | 描述 |
|--------|------|
| 深蓝渐变Banner | 页面顶部展示深蓝渐变背景的Banner区域，突出平台品牌形象 |
| 轮播推荐 | 中部轮播区循环展示推荐书籍（封面+标题+简介），支持自动切换和手动切换 |
| 新书上架 | 展示最新上架的小说列表，采用横向拖拽滚动方式展示，提升交互体验 |
| 热门榜单 | 提供周榜、月榜、总榜三种Tab切换维度，每种榜单展示排名靠前的书籍列表 |
| 书单推荐 | 采用主推大卡片加5张网格的组合布局方式展示精选书单 |

**核心页面元素**：低饱和度极简三色配色方案，#fafbfc浅灰底色，system-ui字体栈确保中文显示正常。

### 2.2 书库模块

提供书籍分类筛选和列表浏览功能，采用金色主题设计（#CA8A04 accent色）配合暖纸质感。

**功能清单**：

| 功能名 | 描述 |
|--------|------|
| 侧边栏分类筛选 | 左侧固定侧边栏展示12个书籍分类，点击分类筛选对应书籍列表 |
| 药丸标签 | 分类标签以药丸形状展示，选中状态高亮显示 |
| 增强书籍卡片 | 书籍卡片支持封面overlay效果和hover上浮动画，提升视觉层次感 |
| 搜索功能 | 支持按书名或作者关键词进行全文模糊搜索 |
| 响应式适配 | PC端多列网格布局，平板减少列数，手机端单列加汉堡菜单 |

### 2.3 详情页模块

全面展示单本小说的信息，帮助用户决定是否开始阅读或进行社交互动。

**功能清单**：

| 功能名 | 描述 |
|--------|------|
| Hero封面区 | 页面顶部大尺寸封面展示区域，配合渐变背景营造视觉冲击力 |
| 数据卡片 | 展示字数统计、阅读量、章节数量、作者信息等关键数据指标 |
| 统一按钮组 | 开始阅读、收藏、关注三个按钮统一采用38px规格高度，保持视觉一致性 |
| 横向章节目录 | 章节列表采用横向flow按钮布局方式展示，紧凑且易于浏览 |
| 评分展示 | 展示星级评分图标和平均分数，直观呈现读者评价 |
| 星级评分弹窗 | 用户点击评分时弹出el-dialog弹窗，支持1-5星打分操作 |
| 书评功能 | 展示读者对本书的评论列表，支持查看评论内容和评论者信息 |

### 2.4 关注作者模块

基于UserFollow模型实现的作者关注功能，构建读者与作者之间的社交连接。

**功能清单**：

| 功能名 | 描述 | API接口 |
|--------|------|---------|
| 关注作者 | 用户点击关注按钮关注指定作者 | follow |
| 取消关注 | 已关注用户可取消对作者的关注 | unfold |
| 关注状态检查 | 检查当前用户是否已关注某位作者 | check |
| 我的关注列表 | 展示当前用户关注的全部作者列表 | myFollows |
| 粉丝列表 | 展示关注当前作者的粉丝列表 | followers |
| 作者详情 | 展示作者的详细信息和作品列表 | authorDetail |

**关联页面**：AuthorProfile.vue（作者主页）、MyFollows.vue（我的关注页）。

### 2.5 评分系统模块

基于NovelRating模型实现的小说星级评分功能。

**功能清单**：

| 功能名 | 描述 | API接口 |
|--------|------|---------|
| 提交评分 | 用户对小说进行1-5星整数评分 | submit |
| 评分统计 | 计算并返回某本小说的平均分和评分人数 | stat |
| 我的评分 | 查询当前用户对某本小说的已有评分记录 | myRating |

**业务规则**：同一用户对同一小说仅保留一条评分记录，重复提交走更新逻辑；平均分实时计算更新；评分修改后同步刷新统计结果。

### 2.6 签到会员模块

提供每日签到和会员权益展示功能，增强用户日活粘性。

**功能清单**：

| 功能名 | 描述 |
|--------|------|
| 每日签到 | CheckIn.vue页面，用户完成每日签到操作获取积分或权益 |
| 会员权益 | Membership.vue页面展示会员等级和对应的专属权益说明 |

**数据支撑**：checkin表记录用户签到数据，membership表记录会员等级和权益配置信息。

### 2.7 个人中心模块

提供用户个人信息管理和阅读数据追踪功能。

**功能清单**：

| 功能名 | 描述 |
|--------|------|
| 收藏管理 | 展示和管理用户收藏的小说列表，支持取消收藏操作 |
| 阅读进度 | 展示用户的最近阅读记录和每本书的阅读进度 |
| 个人信息 | 展示和编辑用户基本信息，包含用户名、头像、邮箱、手机号等字段 |

### 2.8 管理端模块

供管理员使用的综合管理平台，涵盖内容运营和用户管理的核心功能域。

**功能清单**：

| 功能名 | 描述 |
|--------|------|
| 图书管理 | 对全站81本小说进行查看、编辑、下架等管理操作 |
| 分类管理 | 维护12个书籍分类的增删改查操作 |
| 订单管理 | OrderManage页面，查看和处理用户订单数据 |
| 签到管理 | CheckInManage页面，管理用户签到数据和异常处理 |
| 粉丝列表 | AdminFollowers页面，查看平台内用户关注关系的全局数据 |

## 三、非功能需求

### 3.1 性能要求

- 页面首次加载时间应控制在3秒以内（首屏内容可交互）
- 首页轮播组件需正确处理keep-alive缓存，通过onActivated生命周期重新初始化数据
- 书库列表分页加载流畅，增强卡片hover上浮动画不卡顿
- 详情页Hero区域和章节目录渲染响应迅速
- 评分提交后平均分实时更新无感知延迟
- 搜索结果返回时间小于1秒（当前2381章数据规模下）

### 3.2 兼容性要求

- 浏览器：Chrome、Firefox、Edge、Safari的较新版本
- 屏幕分辨率：PC端1920×1080及以上桌面分辨率；响应式适配1366×768笔记本屏幕
- 移动端：手机单列布局配合汉堡菜单导航
- 字体：使用system-ui字体栈，移除Google Fonts引用避免文字模糊问题

### 3.3 安全要求

- Session-based Cookie withCredentials认证机制，跨域请求携带凭证
- 用户密码必须加密存储
- 所有API接口需验证用户身份和权限
- 评分接口防刷：校验范围1-5星整数，同用户同小说唯一约束
- 关注接口幂等：重复关注不创建重复记录
- 防止XSS攻击：用户提交内容进行转义或过滤
- 敏感操作需记录日志

### 3.4 易用性要求

- 界面布局：首页深蓝主题（#2563EB主色/#fafbfc底色）和书库金色主题（#CA8A04 accent/暖纸质感）双体系设计
- 组件规范：统一圆角8px、阴影效果、间距规范
- 按钮规格：详情页核心按钮统一38px高度
- 交互反馈：按钮点击有状态变化，异步操作有loading提示
- 错误提示：接口调用失败时弹出友好提示，不暴露技术细节
- 表单验证：输入框实时校验，错误信息在字段下方显示
