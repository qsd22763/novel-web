# 接口定义文档

## 墨香书阁小说阅读平台

---

## 一、接口规范

### 1.1 基本规范
| 项目 | 规范 |
|------|------|
| 协议 | HTTP/HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 认证方式 | Session (可选Token) |

### 1.2 请求规范
```javascript
// 请求头
{
  "Content-Type": "application/json",
  "Accept": "application/json"
}

// 分页参数
{
  "page": 1,       // 页码，默认1
  "page_size": 20  // 每页数量，默认20
}
```

### 1.3 响应规范
```javascript
// 成功响应
{
  "code": 200,
  "message": "success",
  "data": { ... }
}

// 分页响应
{
  "code": 200,
  "message": "success",
  "data": {
    "count": 100,      // 总数
    "page": 1,         // 当前页
    "page_size": 20,   // 每页数量
    "results": [ ... ] // 数据列表
  }
}

// 错误响应
{
  "code": 400,
  "message": "错误描述",
  "data": null
}
```

### 1.4 状态码
| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 四、用户接口

### 4.1 用户注册
```
POST /api/auth/register/
```

**请求参数**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名(6-20位) |
| password | string | 是 | 密码(6-20位) |
| email | string | 否 | 邮箱 |

**请求示例**
```json
{
  "username": "reader001",
  "password": "password123",
  "email": "reader@example.com"
}
```

**响应示例**
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "reader001"
  }
}
```

### 4.2 用户登录
```
POST /api/auth/login/
```

**请求参数**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

**响应示例**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "id": 1,
```

### 4.3 用户登出

## 九、作者创作接口

> 全部接口都需要登录，且只能操作当前用户名下的作品和章节。

### 9.1 我的作品列表
```
GET /api/author/novels/
```

**响应示例**
```json
{
  "count": 2,
  "results": [
    {
      "id": 12,
      "title": "山河志",
      "author": "墨白",
      "cover": "",
      "category": "玄幻",
      "tags": "热血,东方",
      "status": 0,
      "status_display": "连载中",
      "audit_status": 2,
      "audit_status_display": "已发布",
      "word_count": 12530,
      "view_count": 0,
      "chapter_count": 5,
      "updated_at": "2026-05-29T10:00:00Z"
    }
  ]
}
```

### 9.2 创建作品
```
POST /api/author/novels/
```

**请求参数**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 书名 |
| category | string | 是 | 分类 |
| description | string | 是 | 简介 |
| cover | string | 否 | 封面 URL |
| tags | string | 否 | 标签，逗号分隔 |
| status | int | 否 | 0连载/1完结，默认 0 |

**说明**：`author` 字段由后端使用当前用户的笔名/用户名自动填充；`author_user` 自动绑定当前用户；首次创建时自动把用户的 `is_author` 置为 `True`。

### 9.3 修改作品
```
PATCH /api/author/novels/{id}/
```
请求体支持部分字段更新，参数同 9.2。

### 9.4 删除作品
```
DELETE /api/author/novels/{id}/
```

### 9.5 作品发布 / 下架
```
POST /api/author/novels/{id}/publish/        # 切到已发布
POST /api/author/novels/{id}/take_down/      # 切到已下架
```

### 9.6 作者数据看板
```
GET /api/author/novels/stats/
```
**响应示例**
```json
{
  "total_works": 3,
  "total_chapters": 47,
  "total_words": 158420,
  "total_views": 23901
}
```

### 9.7 章节列表（按作品过滤）
```
GET /api/author/chapters/?novel={novel_id}
```
返回该作品全部章节（包含草稿）。

### 9.8 创建章节
```
POST /api/author/chapters/
```

**请求参数**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| novel | int | 是 | 所属作品 ID（必须为本人作品） |
| title | string | 是 | 章节标题 |
| content | string | 是 | 章节正文 |
| chapter_order | int | 否 | 章节序号，缺省自动取最大值+1 |
| publish_status | int | 否 | 0草稿/1已发布，默认 1 |

**说明**：`word_count` 由后端按非空白字符长度自动计算，并回写到所属作品的 `word_count`。

### 9.9 修改/删除/发布章节
```
PATCH  /api/author/chapters/{id}/
DELETE /api/author/chapters/{id}/
POST   /api/author/chapters/{id}/publish/
```
修改章节时同样会重新计算字数并同步到作品总字数。

### 9.10 越权与可见性
- 阅读端 `/api/novels/`、`/api/novels/{id}/chapters/` 默认只返回 `audit_status=已发布` 的作品和 `publish_status=已发布` 的章节，作者草稿不会泄漏给读者。
- 作者接口在 ViewSet 层使用 `Novel.objects.filter(author_user=request.user)` 过滤，确保无法越权访问他人作品。
