# 项目质量计划书

## 墨香书阁 (Mo Xiang Book Pavilion) V3.0.0

---

## 一、引言

本计划书制定项目的质量目标、管理措施和控制方法，作为开发过程中质量保障的统一依据。适用于墨香书阁V3.0.0版本从启动到交付的全过程质量管理工作，时间跨度为项目开发周期。覆盖功能实现、文档编制和测试验证三个主要质量领域。本项目为企业级高品质小说阅读网站，采用前后端分离架构，前端基于Vue 3 + TypeScript + Vite + Element Plus技术栈，后端基于Python 3.8 + Django 4.2 + DRF框架，数据库使用MySQL。

---

## 二、质量目标

项目的总体质量目标是：交付一个功能完整、运行稳定、体验良好、代码质量过硬的企业级高品质小说阅读平台。

具体目标为：

- **功能完整**：首页、书库、详情、排行榜、搜索、作者主页、我的关注、签到中心、会员中心、用户中心等核心页面及6个后台管理页面均可正常使用，通过逐项验收
- **运行稳定**：核心阅读流程不出现崩溃性错误，TypeScript编译零错误，异常情况有清晰的提示和恢复路径
- **体验良好**：全站UI统一设计语言（首页深蓝主题/书库详情金色主题），界面直观美观，响应式适配PC/平板/手机三断点，用户无需专门培训即可上手操作
- **便于维护**：前后端分离架构清晰，模块之间相互独立，新增功能时不影响已有模块的正常运行，9项Bug全部修复并验证通过
- **数据完整**：81本小说、2381个章节、12个分类、3张新表（user_follow/novel_rating/checkin+membership）数据结构合理，关联关系正确
- **文档齐全**：所有核心文档内容完整、版本号统一为v3.0.0、各文档之间信息一致

## 三、质量保证组织

项目团队需明确质量职责：

- **项目负责人**：负责需求审核、里程碑验收和最终质量决策
- **前端开发**：在Vue 3 + TypeScript组件实现过程中进行自我检查，确保TypeScript编译无错误，不偏离需求和规范
- **后端开发**：在Django API开发过程中进行自我检查，确保接口安全、数据一致性和迁移脚本正确执行
- **测试人员（兼任）**：按测试清单执行人工验证，记录测试结果，重点验证9项Bug修复的有效性

质量活动融入日常开发节奏：每次修改后进行自查和TypeScript编译验证，每个版本完成后进行集中测试和跨浏览器验证（PC/平板/手机），每个主要版本发布前核对全部文档。

## 四、质量标准

### 4.1 功能质量标准

- 首页展示推荐轮播、分类导航、排行榜入口等信息，深蓝色设计主题统一协调
- 书库页支持分类筛选、多维度排序、分页加载，金色设计主题与详情页保持一致

- 排行榜支持多种排行维度切换，数据从后端API动态获取并正确展示
- 搜索功能支持关键词搜索、结果展示和筛选排序
- 作者主页展示作者信息和作品列表
- 我的关注页面实现关注/取消关注功能，数据同步到user_follow表
- 签到中心实现每日签到功能，数据记录在checkin表中
- 会员中心展示会员权益和充值入口，订单记录存储在membership相关表中

- 后台管理包含6个管理页面：书籍管理、分类管理、粉丝管理、签到管理、订单管理等，支持高级筛选和批量操作
- 所有API接口返回正确的HTTP状态码和结构化JSON数据
- 响应式布局在PC端（≥1200px）、平板端（768px-1199px）、手机端（<768px）均正常显示
- TypeScript代码通过严格类型检查，编译零错误

## 五、质量控制活动

**需求阶段**：开发前通读需求文档和Agent规范，确认对V3.0.0新增功能（关注系统、评分系统、签到系统、会员系统）的理解一致。核对3张新表的设计方案是否满足业务需求。

**开发阶段**：每次修改后自查是否违反开发规范；强制使用request.ts工具禁止原生fetch；每次开发任务完成后更新开发日志；每次代码提交前必须执行TypeScript编译检查（npx vue-tsc --noEmit），确保零错误；完成数据库迁移后验证0014_add_checkin_membership、0015_userfollow、0016_novelrating三个迁移文件执行成功且数据完整。

**测试阶段**：按测试清单逐项手动验证；在多种浏览器下检查核心功能；在PC/平板/手机三种设备尺寸下验证响应式布局效果；验证用户认证和授权机制正常工作；逐一验证9项Bug修复的有效性；特别测试关注/取消关注、小说评分、每日签到、会员购买等新增功能的完整流程。

**交付阶段**：交叉核对各文档版本号是否统一为v3.0.0；确认功能列表已覆盖最新版本的全部新增内容（关注/评分/签到/会员）；为版本创建清晰的Git标签；确认代码已提交GitHub并通过编译验证；核对14份文档的内容一致性和时效性。

## 七、质量改进机制

项目建立以下质量改进机制：

- 测试或使用中发现的任何问题均记录在案，标注发现时间和严重程度，V3.0.0已修复9项Bug并全部验证通过
- 问题修复后在开发日志中简要说明原因和措施，形成可追溯的质量改进记录
- 每次版本发布后回顾是否有遗漏的质量检查项，补充到检查清单中
- 文档与代码的同步规则：版本升级后强制核对相关文档，防止内容滞后，V3.0.0已确保14份文档版本统一
- 踩坑经验及时沉淀到Agent行为规则中（如fetch卡死问题→禁止原生fetch规则、TypeScript类型检查→强制npx vue-tsc --noEmit规则）
- TypeScript编译零错误作为代码提交的前置门槛，从制度上保障代码质量
- 响应式适配作为UI验收的标准流程，确保PC/平板/手机三端体验一致
- 定期回顾质量目标的达成情况，持续优化质量管理流程
